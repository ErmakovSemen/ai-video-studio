## Найденные баги

<!-- заполняется по ходу прогона -->

### Шаг 0 — вход и дашборд
- 🟡 Вход: открыл `/` → сразу попал в `/cabinet` без формы логина (сессия закэширована в браузере). Для первого пользователя неясно, как логиниться; повторно не проверить «как новичок» без чистого профиля.
- 🔴 Дашборд → «Статус подключений»: **YouTube Shorts = не подключён** (прочерк), хотя автопостинг в YouTube реально работает (через GitHub Action с YT-секретами). Для пользователя противоречие: канал постит, а UI говорит «не подключён». Публикация с доски в YouTube, скорее всего, не сработает (нет YT-кредов в Render env). Telegram = ✅.

### Шаг 1 — открыть Контент-доску
- 🟠 «Контент» в кабинете (`/cabinet`, iframe) **не загружается** — серая область с иконкой сломанного контента (краш/ошибка фрейма). При этом доска по прямому URL `/board?name=content` работает отлично. Баг встраивания iframe (src/ошибка/таймаут free-инстанса). Пользователь, кликнув «Контент» в меню, видит пустой экран и думает, что всё сломано.
- Обходной путь: открывать доску напрямую `/board?name=content`.

### Шаг 2 — карточка-идея, действия
- 🟡 На стадии «Идеи» у карточки со сценарием НЕТ кнопки «🎬 Сгенерировать видео» (она появляется только в «Драфт/Монтаж»). Путь идея→видео не очевиден: надо сперва перетащить/сменить стадию на «Драфт». Новичок застрянет.
- 🟠 Путаница названий: кнопка «🤖 ИИ-монтажёр» / «Запустить ИИ-монтаж» на самом деле улучшает СУБТИТРЫ, а не монтирует. Рядом колонка «Монтаж» и кнопка «Отдать Claude», и «Профи ИИ-монтаж из сырья». Три разных «монтажа» — пользователь не понимает, что где.
- 🔵 «Отдать Claude» есть и работает как кнопка (без drag) — хорошо. Но подпись «обрабатывается фоном (studio_ctl claim)» утечка внутренней терминологии в UI: пользователю «studio_ctl claim» ничего не говорит.

### Шаг 3 — перевод идеи в «Драфт» (drag)
- 🟡 Смена стадии через дропдаун «Стадия» в карточке нативным селектом неудобна (на macOS OS-нативный список, с клавиатуры значение не сменилось с первого раза). Основной способ — drag-drop.
- 🟡 Drag-drop сработал только со второй попытки (нужен точный захват тела карточки; первый драг «промахнулся»). Для пользователя на тачпаде может быть нестабильно.
- 🟠 Есть кнопка «💾 Сохранить» — изменения доски (перемещения, правки) НЕ автосохраняются. Легко потерять работу, забыв нажать. Ожидание: канбан сохраняется сам.

### Шаг 4 — генерация видео (Драфт)
- ✅ РАБОТАЕТ: кнопка «Сгенерировать видео» на live free-Render отрендерила космос (draft+polish, стиллы+музыка) за ~3.5 мин, видео прикрепилось, карточка авто-переехала в «Монтаж». Инстанс не умер (health 200 во время рендера).
- 🟡 После завершения модалка не обновляется сама: всё ещё показывает «Видео ещё нет» и кнопку «Сгенерировать», статус-строка даёт ссылку «обновить» вручную. Ожидание: карточка/превью обновятся автоматически.
- 🟡 Долгий рендер (~3.5 мин) без прогресс-бара — только «running». На free-CPU пользователь не понимает, завис или работает; нужен прогресс/ETA.
- 🔵 Chrome-расширение один раз отвалилось во время ожидания (среда автоматизации, не баг продукта) — но напоминание: длинные операции стоит делать устойчивыми к перезагрузке вкладки (джоб виден по reopen — это ок, статус сохранился на сервере).

### Шаг 5 — артефакт видео и durability
- 🟠 На карточку сохраняется ЭФЕМЕРНЫЙ путь `/outputs/<jid>.mp4` (виден в поле артефакта), а НЕ durable catbox-URL — хотя джоб рендера его отдаёт (`url`). После засыпания/перезапуска free-инстанса ссылка на видео умрёт, карточка «потеряет» ролик. board.html (m_gen) должен писать `data.url` (catbox), а не `/outputs/...`.
- 🟡 Видеоплеер в карточке грузится медленно (free-инстанс отдаёт /outputs) — крутилка несколько секунд.

### Шаг 6 — постинг (ФИНАЛ, блокер для YouTube)
- 🔴 ГЛАВНЫЙ БЛОКЕР пути «до YouTube»: на live-инстансе `/api/publishers` → **youtube.configured = false** (нет YT client_id/secret/refresh_token в Render env). Telegram.configured = true. Значит кнопка «📤 Запостить» с доски уйдёт ТОЛЬКО в Telegram; залить на YouTube со страницы НЕЛЬЗЯ.
- Причина: YT-секреты живут в GitHub Secrets (для autopost-Action), а в окружении Render-страницы их нет. Рабочая заливка в YouTube идёт ТОЛЬКО через GitHub Action `autopost_daily` (проверено ранее: youtu.be/JrxyU62HcJM), а не через доску.
- 🔵 Чтобы доска умела в YouTube: либо добавить YT_* в Render env (тогда «Запостить» заработает в YT), либо в UI честно показывать «YouTube постится автоматически по расписанию, ручной постинг недоступен».
- 🟡 Нет шага «отдать на YouTube по расписанию» из карточки: пользователь, дойдя до «Залито», не понимает, что YouTube идёт отдельным автопостингом, а не этой кнопкой.

### Шаг 7 — навигация между колонками
- 🟡 Колонки «Ревью / Ждёт постинга / Постинг / Залито» уходят за правый край (горизонтальный скролл). Перетаскивать карточку через весь экран в дальнюю колонку неудобно; смены стадии через селект — ненадёжны. Нужен «следующий шаг»-кнопкой прямо на карточке (частично есть в Ревью, но не везде).

# Переезд с Render на Beget VPS

Цель: постоянный сервер, который не засыпает и тянет тяжёлую обработку
(whisper-транскрибация, ffmpeg-рендер, генерация картинок) — в отличие от
Render free-tier (512MB, засыпает). Beget выбран из-за нормальной оплаты
из России (карта РФ / крипта), без гео-блоков как у Oracle/Hetzner.

## Шаг 1 — заказ сервера (руками, это твой шаг)

1. https://beget.com/ru/vps → Заказать VPS/VDS.
2. **Локация — выбирай европейскую**, если предложена (у Beget с 2026 есть EU-локации),
   а не российскую. Причина: сервер будет ходить в OpenRouter/Google/YouTube API —
   EU-датацентр надёжнее для этого трафика, чем RU-IP.
3. **Тариф — минимум 4GB RAM** (лучше 6GB): whisper-модель + ffmpeg-рендер +
   параллельные запросы к OpenRouter требуют памяти. Смотри тариф "6 GB"
   (4 ядра, 80GB NVMe, ~2190₽/мес) — с запасом, либо ближайший вариант с 4GB.
4. ОС — **Ubuntu 22.04**.
5. Способ входа — SSH-ключ (если Beget предлагает при заказе — вставь свой
   публичный ключ; если только пароль root — тоже нормально, поменяем на ключ
   после первого входа).
6. Оплати (карта РФ или крипта — как удобнее).

## Шаг 2 — когда сервер выдан, пришли мне

- IP-адрес сервера
- root-пароль (или подтверди SSH-ключ) — **не пиши его в общий чат, если это чувствительно;
  можно прислать одноразово, я сразу переведу вход на ключ и сменю пароль**

Дальше сделаю сам по SSH:
- обновлю систему, поставлю Docker + docker compose
- залью `Dockerfile` / `docker-compose.yml` (уже готовы в репо)
- перенесу секреты (`client_secret.json`, `yt_token.json`, `config/credentials.json`)
  с локальной машины через `scp` — не через чат
- заведу `.env.prod` из `.env.prod.example` со значениями из `.secrets.env`
- подниму Caddy для авто-TLS (если заведёшь A-запись домена на IP сервера —
  тогда сразу будет `https://`; без домена — временно по IP:8000)
- настрою `docker compose up -d` с `restart: unless-stopped` — переживает
  перезагрузку сервера
- переключу GitHub Actions `deploy.yml` (сейчас дёргает Render Deploy Hook) на
  деплой через SSH на Beget-сервер
- закрою root-вход по паролю, оставлю только SSH-ключ (базовая защита)

## Что уже готово в репозитории

- `Dockerfile` — python:3.12-slim + ffmpeg.
- `.dockerignore` — секреты/venv/outputs не попадают в образ.
- `docker-compose.yml` — монтирует `data/` (секреты, outputs, media) снаружи образа.
- `.env.prod.example` — список нужных переменных окружения (без значений).

Всё это провайдеро-независимо (обычный Ubuntu + Docker) — не привязано к Oracle,
подойдёт для Beget без изменений.

## Домен (опционально, но лучше сделать)

Если хочешь `https://` без ворнинга браузера — нужен домен (даже бесплатный
поддомен). Направляем A-запись на IP Beget-сервера, Caddy сам получит
сертификат Let's Encrypt. Без домена — просто `http://<IP>:8000`, басик-авторизация
пойдёт открытым текстом, что не супер для продакшена.
